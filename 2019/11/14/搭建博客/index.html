搭建博客

Posted on 2019-11-14 | Post modified: 2019-11-14 |

博客搭建基于 GitHub + Hexo.

准备环境

在安装Hexo之前需要先安装下列应用程序：
Node.js
Git

GitHub创建个人仓库

如果没有github账号，在GitHub完成注册。
接着创建Repository: 登录GitHub，点击有右上角的 + ,选择New repository,
Repository name的固定格式为：username.github.iousername就是你的github用户名,其余的可以先不填，点击Create repository。

配置和使用GitHub

点击开始 → Git → Git Bash

配置SSH Keys

通过SSH Keys可以与GitHub建立联系，更新博客内容。

  • 检查本地是否有SSH Keys：$ cd ~/.ssh (在打开的 Git Bash 中输入)
    如果显示：No such file or directory,则表示这是第一次使用Git。
  • 生成新的SSH Keys
$ ssh-keygen -t rsa -C "邮箱地址"           #邮箱地址为注册GitHub的邮箱地址。
Generating public/private rsa key pair.
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>

然后需要输入密码（这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容）。

Enter passphrase (empty for no passphrase):    #<设置密码>
Enter same passphrase again:     #<再次输入密码> 
#输入密码时，没有什么都没有显示，但是是在输入的。

  • 添加SSH Keys到GitHub
    在本地文件夹找到id_rsa.pub文件，用记事本打开该文件，全选复制。
    打开SSH Keys设置页面,点击New SSH Keys,填写title，粘贴刚才的内容到Key中。点击Add SSH Key。

  • 测试
    在Git Bash中输入：$ ssh -T git@github.com,
    回车，会要求输入密码Enter passphrase for key '/c/Users/Lee/.ssh/id_rsa':
    输入密码，回车，看到Hi xxxx! You've successfully authenticated, but GitHub does not provide shell access.
    则表示配置成功。

设置用户信息

$ git config --global user.name    #注册GitHub的用户名
$ git config --global user.email   #注册GitHub的邮箱

搭建Hexo博客

  • 安装Hexo
$ npm install -g hexo-cli
  • 在本地创建项目文件夹
    在本地创建一个与GitHub上的Repository同名的文件夹username.github.io。
    文件夹 → 右键 → 选择Git Bash Here(在搭建博客工作时，每次使用命令，都要在username.github.io目录下）。
$ hexo init #初始化，初始化后会有一个默认的主题：landscape，和一篇自动生成的hello world的文章

$ hexo g/generate  #生成

$ hexo s/server  #启动服务预览

接着打开浏览器，输入：localhost:4000就可以看到了。

将博客部署到GitHub

  • 安装相关插件
$ npm install hexo-deployer-git --save
  • 编辑博客配置文件

本地打开username.github.io/_config.yml,修改配置文件：

deploy
  type: git
  repository: git@github.com:username/username.github.io.git   #username为github用户名
  branch: master
  • 部署
$ hexo g
$ hexo d/deploy    #部署(如果有设置密码，需要输入密码)

接着，在浏览器中输入username.github.io，就可以看到自己的博客了。

写博客

  • 新建文章
$ hexo new [layout] <title>    #title中有空格的话用 "" 括起来

例

$ hexo new "my first blog"

[layout]可以为：

参数名 功能 路径
post 新建博文 source/_posts
page 新建页面 source
draft 新建草稿 source/_drafts

hexo new <title>创建一篇新文章（[layout]默认为post类型），在username.github.io\source\_posts中可以看到新创建的title.md文件。打开就可以对文章进行编辑，markdown语法可参考：markdown语法

  • 新建草稿
$ hexo new draft <title>
  • 本机预览草稿
$ hexo s --draft

打开localhost:4000 就可以预览草稿了

  • 发布草稿
$ hexo publish/p <title>
  • 删除文章
    直接在本地删除.md文件
$ hexo d -g  #更新推送

绑定域名

以腾讯云为例。

  • 购买域名。可以在腾讯云，阿里云或者GoDaddy等网站上购买。
  • 实名认证。超过有效期（3-5天）没有实名认证，域名会被锁定。
  • 域名解析。实名认证成功后的域名，才能被解析。
    打开我的域名,点击解析 → 添加记录。
    解析设置添加两条A记录，一条CNAME记录：
    • CNAME:主机记录选www,记录类型选CNAME,线路选默认,记录值填Github博客的网址(username.github.io)。点击保存。
    • A:主机记录选@,记录类型选A,线路选默认,A记录的就是ip地址，记录值有两个，所以要添加两个@主机记录，记录值分别为：192.30.252.153和192.30.252.154。
    • 设置自定义域
      登录GitHub，进入username.github.io仓库，点击settings，设置Custom domain，输入域名
    • 本地设置
      在username.github.io\node_modules\hexo根目录的source文件夹里创建CNAME文件，不带任何后缀，里面添加你的域名信息，如：name.com。如果填写的是www.name.com那么只能用www.name.com访问，而如果填写的是 name.com ，那么用www.name.com和name.com访问都是可以的。
    $ hexo d -g
    更新后设置就可以用新的域名进行访问了。
  • 备案。如果服务器是国外的，就不需要备案。像GitHub服务器就不需要备案了。

博客个性化

更改站点信息

打开username.github.io/_config.yml文件，可修改信息。

# Site
title:                        #网站标题
subtitle:                     #网站副标题
description:                  #网站描述(主要用于SEO，告诉搜索引擎一个关于您站点的简单描述，通常建议在其中包含您网站的关键词。)
keywords:                     #网站的关键词。使用半角逗号 , 分隔多个关键词。
author:                       #用于主题显示文章的作者
language: zh-CN               #网站使用的语言
timezone:                     #网站时区。默认使用您电脑的时区。

更改主题

  • 进入Hexo官网主题，找到喜欢的主题后，进入github页面 →点击 Clone or download →复制链接，
    接着复制到本地themes文件下：
    右键username.github.io文件夹→ 选择Git Bash Here
    输入 git clone 复制的链接 themes/主题名

  • 本地设置
    打开username.github.io文件夹下的_config.yml文件,找到#Extensions,把默认主题landscape改为新下载的主题名。

  • 然后在Git Bash中输入：

$ hexo g
$ hexo d  #部署到github
$ hexo s
  • 接着打开浏览器，输入：localhost:4000就可以看到新主题。

添加评论模块

Disqus(需要科学上网)

  • 注册Disqus
  • 配置Disqus
    点击Get Start → I want to install Disqus on my site

Website Name :自己填写.
Category 选择种类，可以随便选
Language 选 Chinese 或者 English
点击 Create Site
Select Plan 选择 Basic
Install Disqus 点击 I don't see my platform listed,install manually with Univeral Code → Configure →填写Website URL → 点击Complete Setup

  • 本地配置
    打开username.github.io/next/_config.yml
    找到Disqus，配置：
# Disqus
disqus:
  enable: true
  shortname:           #输入刚才的 Website Name
  count: true

完成。

主页文章添加边框阴影效果

打开themes/*/source/css/_custom/custom.styl,添加代码：

.post {
   margin-top: 0px;
   margin-bottom: 60px;
   padding: 25px;
   -webkit-box-shadow: 0 0 5px rgba(202, 203, 203, .5);
   -moz-box-shadow: 0 0 5px rgba(202, 203, 204, .5);
}

开启RSS订阅服务

  • 安装RSS插件
$ npm install hexo-generator-feed --save
  • 修改 hexo 配置文件username.github.io/_config.yml:
#RSS 订阅
feed: 
    type: atom                 #RSS的类型(atom/rss2)
    path: atom.xml             #文件路径,默认是atom.xml/rss2.xml
    limit: 20                  #展示文章的数量,使用0或则false代表展示全部
    hub:
    content:                   #在RSS文件中是否包含内容 ,有3个值 true/false默认不填为false
    content_limit:             #指定内容的长度作为摘要,仅仅在上面content设置为false和没有自定义的描述出现
    content_limit_delim: ' '   #上面截取描述的分隔符,截取内容是以指定的这个分隔符作为截取结束的标志.在达到规定的内容长度之前最后出现的这个分隔符之前的内容,，防止从中间截断

设定菜单项的文件目录和对应图标

修改对应图标

打开 themes/*/_config.yml 文件，搜索 menu_icons 关键字，修改对应图标名称或者新增对应 menu 的图标。

menu:
  home: /||home
  about: /about/ ||user
  tags: /tags/ || tags 
  #categories: /categories/ || th
  archives: /archives/|| archive
  #schedule: /schedule/ || calendar
  #sitemap: /sitemap.xml || sitemap
  commonweal: /404/ || heartbeat

# Enable/Disable menu icons.
menu_icons:
  enable: true

除了 home， archives , /后面都需要手动创建这个页面

创建菜单项对应文件目录

  • 创建404页面
$ hexo new page 404

在username.github.io\source\404中可以看到新创建的index.md文件。
打开文件，在顶部插入permalink: /404，指定该页固定链接为 https://home/404.html

---
title: 404
date: 2019-10-22 10:25:56
comments: false
permalink: /404
---

页面内容可以自定义，也可以设置成腾讯公益404界面：

<!DOCTYPE HTML>
<html>
<head>
  <meta http-equiv="content-type" content="text/html;charset=utf-8;"/>
  <meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1" />
  <meta name="robots" content="all" />
  <meta name="robots" content="index,follow"/>
  <link rel="stylesheet" type="text/css" href="https://qzone.qq.com/gy/404/style/404style.css">
</head>
<body>
  <script type="text/plain" src="http://www.qq.com/404/search_children.js"
          charset="utf-8" homePageUrl="/"
          homePageName="回到我的主页">
  </script>
  <script src="https://qzone.qq.com/gy/404/data.js" charset="utf-8"></script>
  <script src="https://qzone.qq.com/gy/404/page.js" charset="utf-8"></script>
</body>
</html>
  • 创建分类页面
$ hexo new page categories

打开刚新建的页面,修改Front-matter：

---
title: 分类
date: 2014-12-22 12:39:04
categories: name #分类名
type: "categories"
---

Front-matter 是文件最上方以 --- 分隔的区域，用于指定个别文件的变量

  • 创建标签页面
$ hexo new page tags

打开刚新建的页面,修改Front-matter：

---
title: 分类
date: 2014-12-22 12:39:04
type: "tags"
---

设置网站图标(favicon)

  • 打开favicon制作，设置自己的图标
  • 将图片保存于 Gjal.github.io\themes\next\source\images。
  • 打开username.github.io/next/_config.yml,修改设置：
favicon:
  small: /images/图片名.png        #换成自己的图片名称
  medium: /images/图片名.png       #换成自己的图片名称
  apple_touch_icon: /images/apple-touch-icon-next.png
  safari_pinned_tab: /images/logo.svg
  #android_manifest: /images/manifest.json
  #ms_browserconfig: /images/browserconfig.xml
