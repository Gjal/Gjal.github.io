GitHub创建个人仓库

如果没有github账号，在GitHub完成注册。
接着创建Repository: 登录GitHub，点击有右上角的 + ,选择New repository,
Repository name的固定格式为：username.github.iousername就是你的github用户名,其余的可以先不填，点击Create repository。

配置和使用GitHub

点击开始 → Git → Git Bash

配置SSH Keys

通过SSH Keys可以与GitHub建立联系，更新博客内容。

  • 检查本地是否有SSH Keys：$ cd ~/.ssh (在打开的 Git Bash 中输入)
    如果显示：No such file or directory,则表示这是第一次使用Git。
  • 生成新的SSH Keys

然后需要输入密码（这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容）。

  • 添加SSH Keys到GitHub
    在本地文件夹找到id_rsa.pub文件，用记事本打开该文件，全选复制。
    打开SSH Keys设置页面,点击New SSH Keys,填写title，粘贴刚才的内容到Key中。点击Add SSH Key。

  • 测试
    在Git Bash中输入：$ ssh -T git@github.com,
    回车，会要求输入密码Enter passphrase for key '/c/Users/Lee/.ssh/id_rsa':
    输入密码，回车，看到Hi xxxx! You've successfully authenticated, but GitHub does not provide shell access.
    则表示配置成功。

绑定域名

以腾讯云为例。

  • 购买域名。可以在腾讯云，阿里云或者GoDaddy等网站上购买。
  • 实名认证。超过有效期（3-5天）没有实名认证，域名会被锁定。
  • 域名解析。实名认证成功后的域名，才能被解析。
    打开我的域名,点击解析 → 添加记录。
    解析设置添加两条A记录，一条CNAME记录：
    • CNAME:主机记录选www,记录类型选CNAME,线路选默认,记录值填Github博客的网址(username.github.io)。点击保存。
    • A:主机记录选@,记录类型选A,线路选默认,A记录的就是ip地址，记录值有两个，所以要添加两个@主机记录，记录值分别为：192.30.252.153和192.30.252.154。
    • 设置自定义域
      登录GitHub，进入username.github.io仓库，点击settings，设置Custom domain，输入域名
    • 本地设置
      在username.github.io\node_modules\hexo根目录的source文件夹里创建CNAME文件，不带任何后缀，里面添加你的域名信息，如：name.com。如果填写的是www.name.com那么只能用www.name.com访问，而如果填写的是 name.com ，那么用www.name.com和name.com访问都是可以的。
    更新后设置就可以用新的域名进行访问了。
  • 备案。如果服务器是国外的，就不需要备案。像GitHub服务器就不需要备案了。

遇到的问题

中文乱码

用记事本编辑会乱码
用VScode 编辑就没有乱码(Atom、Sublime Text也可以)
themes-(next)-languages里面没有的包（zh-CN）也不会出现乱码。
只是编辑器的问题。
记事本保存编码格式为“UTF-8" 也可以解决乱码问题，但是需要另存，太麻烦了。

表格不能居中显示

Hexo 默认其官方插件 hexo-renderer-marked 使用 marked 来渲染 Markdown。这个插件有很多特殊 Markdown 语法不支持。Hexo 官方还有另一个使用 markdown-it 做渲染器的插件 hexo-renderer-markdown-it。

  • 安装
    首先需要先删除默认的 hexo-renderer-marked 插件
    npm uninstall hexo-renderer-marked
    再安装 hexo-renderer-markdown-it 插件
# npm install --save hexo-renderer-markdown-it
# 直接从 GitHub 上安装，支持配置插件
npm install --save git+https://github.com/hexojs/hexo-renderer-markdown-it.git
  • 配置
    打开_config.yml,添加如下配置信息:
# Markdown-it config
## Docs: https://github.com/celsomiranda/hexo-renderer-markdown-it/wiki
markdown:
  render:
    # Enable HTML tags in source
    html: true

    # Use '/' to close single tags (<br />). This is only for full CommonMark compatibility.
    xhtmlOut: true        

    # Convert '\n' in paragraphs into <br> 
    breaks: true      

    # CSS language prefix for fenced blocks. Can be useful for external highlighters.
    langPrefix: 'language-'  

    # Autoconvert URL-like text to links 
    linkify: true        

    # Enable some language-neutral replacement + quotes beautification
    typographer: false

    # Double + single quotes replacement pairs, when typographer enabled,
    # and smartquotes on. Could be either a String or an Array.
    #
    # For example, you can use '«»„“' for Russian, '„“‚‘' for German,
    # and ['«\xA0', '\xA0»', '‹\xA0', '\xA0›'] for French (including nbsp).
    quotes: '“”‘’'

  # Plugins
  plugins:
    - markdown-it-abbr
    - markdown-it-footnote
    - markdown-it-ins
    - markdown-it-sub
    - markdown-it-sup
    - markdown-it-anchor
    - markdown-it-deflist
    - markdown-it-mark
    - markdown-it-container

    - markdown-it-emoji
    - markdown-it-attrs
    - name: markdown-it-task-lists
      options:
        enabled: false
        label: true
        labelAfter: false
  
  # Automatic Headline ID's
  anchors:
    # Minimum level for ID creation. (Ex. h2 to h6)
    level: 2

    # A suffix that is prepended to the number given if the ID is repeated.
    collisionSuffix: 'v'           

    # If `true`, creates an anchor tag with a permalink besides the heading.
    permalink: false              

    # Class used for the permalink anchor tag.
    permalinkClass: header-anchor 

    # The symbol used to make the permalink
    permalinkSymbol: ¶
  • 安装插件
npm install --save markdown-it-abbr          #缩写
npm install --save markdown-it-footnote      #脚注
npm install --save markdown-it-ins           #<ins>
npm install --save markdown-it-sub           #下标
npm install --save markdown-it-sup           #上标
npm install --save markdown-it-anchor        #锚点
npm install --save markdown-it-deflist       #定义列表
npm install --save markdown-it-mark          #<mark>
npm install --save markdown-it-container     #自定义容器
npm install --save markdown-it-emoji         #表情
npm install --save markdown-it-attrs         #HTML属性
npm install --save markdown-it-task-lists    #task list
